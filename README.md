# 校内创新创业项目报名系统

Java 控制台轻量程序，模拟校内创新创业项目报名管理。突出 **数据清洗、批量处理、并发校验、缺陷拦截** 核心能力，贴合后端 / 数据测试求职方向。

## 环境要求

- JDK 8+
- sqlite-jdbc (纯 Java 实现，单 JAR，~3MB)

## 快速启动

### 方式一：一键运行（Windows）

```
双击 compile.bat  →  自动下载依赖 + 编译
双击 run.bat      →  启动程序
```

### 方式二：手动命令行

```bash
# 1. 下载 SQLite JDBC 驱动
#    https://repo1.maven.org/maven2/org/xerial/sqlite-jdbc/3.45.1.0/sqlite-jdbc-3.45.1.0.jar

# 2. 编译
javac -cp ".;sqlite-jdbc-3.45.1.0.jar" -encoding UTF-8 RegistrationSystem.java

# 3. 运行
java -cp ".;sqlite-jdbc-3.45.1.0.jar" -Dfile.encoding=UTF-8 RegistrationSystem
```

## 功能模块

| 编号 | 功能 | 说明 |
|------|------|------|
| 1 | 📝 新增报名 | 逐条录入，9 个字段强校验 |
| 2 | 📥 多线程批量导入 | 线程池并发处理 CSV，含重复拦截、校验失败统计 |
| 3 | 🔍 查询报名记录 | 按学号 / 院系 / 全部查询 |
| 4 | 📊 院系统计报表 | 各院系报名人数、占比、数据有效率 |
| 5 | 💾 导出数据 | CSV (Excel兼容) / JSON 双格式导出 |
| 6 | 🤖 量化对比 | 自动化校验 vs 人工核对（耗时/准确率/交付周期） |
| 7 | 📝 操作日志 | 内存 + 数据库双重持久化，含级别统计 |
| 8 | 🗑 重置数据 | 清空报名记录和日志 |

## 数据校验机制

| 校验类型 | 说明 |
|----------|------|
| 空白表单拦截 | 学号/姓名/院系/专业/年级/项目名称 6 个必填字段 |
| 格式校验 | 学号(6-20位字母数字)、姓名(2-20中文英文)、年级(20XX级)、手机号(11位)、邮箱 |
| 院系合法性 | 必须在 12 个合法院系列表中 |
| 重复报名拦截 | 数据库 UNIQUE 约束 + 应用层双重校验 |

## 多线程批量导入

使用 `Executors.newFixedThreadPool(n)` 创建固定线程池，每条 CSV 数据提交为一个任务并发处理。并发场景下的防御措施：

- `synchronized (DB_WRITE_LOCK)` 保护 SQLite 写入
- `CountDownLatch` 等待所有线程完成
- `AtomicInteger` 线程安全计数
- UNIQUE 约束兜底拦截

## 量化对比指标

```
▸ 自动化减少人工核对工作量: >99%
▸ 数据校验错误减少率:       98%
▸ 数据交付周期缩短:         >99%
▸ 人工逐条核对: 5min/条 → 自动化: 0.1s/条
```

## 数据存储

全部数据存储在 SQLite 文件 `registration.db` (WAL 模式)：

- `registrations` — 报名记录 (含学号唯一约束)
- `operation_logs` — 操作日志

## 示例 CSV 数据

`sample_data.csv` 内置 14 条测试数据，覆盖：
- 10 条正常数据（7 个不同院系）
- 1 条空字段数据（触发空白校验）
- 1 条重复学号数据（触发重复拦截）
- 1 条非法院系数据（触发院系校验）
- 1 条非法年级数据（触发格式校验）
