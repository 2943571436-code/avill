> 🎮 零依赖 · 单文件 · 控制台游戏 · SQLite持久化
>
> **面向后端/数据测试求职方向**的交互式作品集项目

---

## 🎯 项目定位

这是一个**融合数据质量检测与闯关玩法**的控制台游戏。你将扮演智能工厂的「数据质量工程师」，在传感器数据流中精准识别异常读数——缺失值、越界值、噪声尖峰、漂移异常、格式错误——守护数据链路的完整性。

关键在于：**这不是单纯的游戏，其底层是一套完整的数据质量检测引擎**——规则校验、缺陷注入、指标计算、SQLite全链路存储。

**核心展示的数据岗能力**：
- **数据校验**（范围检查 / 格式校验 / IQR异常 / 漂移识别 / 噪声检测 — 7种缺陷类型）
- **缺陷注入引擎**（程序化生成带标签的脏数据，可复现、可量化）
- **质量指标**（准确率 / 查全率 / 查准率 / F1-score，完整混淆矩阵）
- **SQLite持久化**（5表：玩家档案 + 关卡记录 + 操作日志 + 传感器数据 + 指标快照）
- **批处理 & 异常捕获**（500+条传感器读数，全流程异常保护）

## 🎮 游戏玩法

### 关卡设计

| 关卡 | 名称 | 传感器 | 读数数 | 限时 | 缺陷类型 |
|------|------|--------|--------|------|----------|
| 1 | 温度巡检 | 温度 | 10 | 60s | 越界、缺失 |
| 2 | 湿度警报 | 湿度 | 12 | 50s | 越界、缺失、格式错误 |
| 3 | 压力测试 | 气压 | 15 | 45s | 越界、噪声尖峰、漂移、负值 |
| 4 | 多传感器融合 | 温度+湿度+振动 | 18 | 55s | 全类型（除零值停滞） |
| 5 | 极限挑战 | 全部5种传感器 | 20 | 50s | 全部7种缺陷类型 |

### 7种数据缺陷类型

| 缺陷类型 | 说明 | 示例 |
|----------|------|------|
| `out_of_range` | 数值越界 | 温度 85°C（正常15~35°C） |
| `missing_value` | 缺失值 | 传感器返回 `None` |
| `format_error` | 格式错误 | 数值字段出现 `"N/A"` `"ERR"` |
| `noise_spike` | 噪声尖峰 | 值瞬间跳变5~12倍 |
| `drift_anomaly` | 漂移异常 | 连续3点单调剧烈漂移 |
| `negative_value` | 负值异常 | 气压 -500 hPa（物理不可行） |
| `zero_reading` | 零值停滞 | 连续4个读数为0（传感器离线） |

### 计分规则

```
基础分   = F1-score × 0.7 + Accuracy × 0.3
时间奖励 = (剩余时间 / 总时限) × 20
误报扣分 = 误报数 × 3
关卡得分 = max(0, 基础分 + 时间奖励 - 误报扣分)

通关标准: F1-score >= 60%
```

### 输出指标（可直接写入简历）

```
准确率(Accuracy) — (TP+TN) / Total    → 综合判断能力
查全率(Recall)   — TP / (TP+FN)       → 缺陷发现率
查准率(Precision)— TP / (TP+FP)       → 标记正确率（误报控制）
F1综合分         — 查全与查准的调和平均
响应耗时         — 单关决策时间(秒)
时间奖励         — 效率加分
```

## 🗃️ SQLite 数据库结构

```
sensor_guardian.db (单文件，零配置)

├── player_profile      玩家档案（累计分数/等级/准确率）
├── level_records       关卡记录（每次挑战的完整指标）
├── sensor_readings     传感器数据（每次生成的读数+缺陷标签）
├── action_log          操作日志（逐条判断的答案对比）
└── metrics_snapshot    指标快照（每局游戏的汇总报告）
```
